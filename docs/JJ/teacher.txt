User Story: Teacher/Facilitator Experience for Quest-Based Learning Platform

Overview

As a teacher/facilitator in a self-directed learning environment, I want to guide students through personalized, interest-led learning journeys so that I can support autonomy, track progress, scaffold learning appropriately, and celebrate growth across projects, portfolios, and community contributions.


---

Epic 1: Student Quest Planning & Management

User Story 1.1 — Create Personalized Learning Quests

As a facilitator, I want to sit with a student one-on-one to identify their interests, strengths, and learning goals so that the platform can generate personalized project-based learning quests.

Acceptance Criteria

Facilitator can create or edit a student profile.

Facilitator can input:

Student interests

Learning goals

Skill level/age cohort

Learning supports/preferences


AI generates suggested project ideas and task lists.

Student and facilitator can:

Accept tasks

Reject tasks

Regenerate alternatives

Add custom tasks manually


Quests remain editable over time.



---

User Story 1.2 — Differentiate AI Task Generation

As a facilitator, I want AI-generated tasks to reflect a student’s developmental and learning needs so that tasks feel accessible and achievable.

Acceptance Criteria

Facilitator can select supports such as:

Smaller task chunks

Quick wins

Minimal reading

More visual/hands-on tasks

Longer-term challenge projects

Independent vs collaborative work


AI adapts:

Vocabulary complexity

Task length

Reading demands

Suggested pacing

Number of steps


AI also considers:

Student age cohort

Previous quest completion history

Skill level



---

User Story 1.3 — Track Multiple Active Projects

As a facilitator, I want students to pause and resume quests while exploring multiple interests so that learning remains flexible and student-driven.

Acceptance Criteria

Students can have multiple active quests.

Quests can be:

Active

Paused

Completed


Students can switch between projects without losing progress.

Progress is automatically saved.

Facilitator dashboard shows:

Current active quests

Paused quests

Completion percentage

Last activity/check-in date

Epic 2: Badge & Pin Quest System

User Story 2.1 — Preloaded Badge-Based Quests for Younger Students

As a facilitator, I want to preload curated quests for younger students so they can choose learning experiences visually and independently.

Acceptance Criteria

Facilitator can create reusable “badge” or “pin” quests.

Each badge includes:

Icon/image

Title

Description

Domain/category

Recommended age group


Younger students can browse quests visually by badge icon.

Students can select quests independently.



---

User Story 2.2 — Categorize Badge Quests

As a facilitator, I want badges categorized by learning domain so students can easily explore areas of interest.

Categories

Wellness

Creative Expression (Art)

STEM

Communication

Civics


Acceptance Criteria

Badges are filterable by category.

Facilitator can create new categories.

Student dashboard supports visual browsing by domain.



---

User Story 2.3 — Completed Pin Tracking

As a facilitator, I want completed quests collected in one location so I can prepare physical pins/badges before celebrations.

Acceptance Criteria

Completed quests automatically move into a “Pins Ready to Create” queue.

Dashboard shows:

Student name

Badge earned

Completion date

Portfolio artifacts/photos


Facilitators can batch mark pins as created/distributed.

Additional Epic: Portfolio Documentation & Observation Journaling

Epic 6: Facilitator Observation & Portfolio Documentation


---

User Story 6.1 — Social-Style Observation Posting

As a facilitator, I want to quickly document meaningful moments of student learning during free exploration so that authentic learning can be captured without interrupting engagement.

Acceptance Criteria

Facilitator can create a journal entry using:

Photos

Video clips (future enhancement)

Written captions/observations


Posting flow feels lightweight and fast, similar to creating a social media post.

Facilitator can:

Tag one or multiple students

Add hashtags/tags/categories

Select related learning domains


One post automatically populates in all tagged students’ portfolios.

Entries are timestamped and organized chronologically.



---

User Story 6.2 — AI-Assisted Portfolio Analysis

As a facilitator, I want AI to analyze my observation captions so that student portfolios can identify patterns of engagement, growth, and skill development without requiring extensive manual documentation.

Acceptance Criteria

AI analyzes:

Caption text

Selected tags/categories

Quest associations

Portfolio domain relevance


AI automatically:

Suggests relevant skill tags

Identifies recurring interests/themes

Maps observations to portfolio domains:

Wellness

Creative Expression

STEM

Communication

Civics


Detects patterns of engagement over time

Suggests evidence of:

Collaboration

Leadership

Problem solving

Creativity

Persistence

Communication skills

Initiative




---

User Story 6.3 — Lightweight Documentation Workflow

As a facilitator, I want documentation to happen naturally during the school day so that portfolio creation does not become administratively overwhelming.

Acceptance Criteria

Journal entries can be created in under 1 minute.

Mobile-friendly interface optimized for quick uploads.

Facilitator can:

Snap photo

Add quick caption

Tag students

Post immediately


Draft saving available for unfinished entries.

Voice-to-text caption support available.



---

As a facilitator, I want portfolios to accumulate authentic evidence of student interests and competencies over time so I can better support learning pathways and showcase growth.

Skill trends over time

As a facilitator, I want AI-generated summaries of student engagement patterns so I can more easily prepare conferences, reports, and personalized guidance.
