Student User Experience (Ages 5–9)

Overview

As a young student, I want a simple, visual, low-reading interface that helps me independently choose projects, track progress, ask for help, and celebrate my learning so that I can confidently participate in self-directed learning without becoming overwhelmed.

(All of this hinges on a child being able to easily access the platform from a shared touchscreen device (widget). If they have to wait for a facilitator's phone, they will move on too quickly. We have a rhythm of catching photos throughout the day of things kids do and uploading them to our portfolio building platform, but we don't have a place where kids set and track goals)

---

Epic 1: Young Learner Login & Navigation Experience

User Story 7.1 — Simple Touchscreen Login

As a young student (ages 5–9), I want to select my profile from a touchscreen device so that I can log in independently without passwords or complicated navigation.

Acceptance Criteria

Platform supports kiosk mode on:

Shared Chromebook

Tablet

Touchscreen devices


Login screen displays:

Student photo

Student first name


No passwords required for young learners.

Interface minimizes distractions:

No avatar creation/customization

No unnecessary menus

Limited text

No access to editing my full portfolio or any account features


Large touch-friendly buttons optimized for young users.

Epic 6: Student Pride Sharing & Facilitator Reflection Support

User Story 6.1 — “I’m Proud of This” Sharing Button

As a young student, I want an easy way to tell my facilitator when I finish something I’m proud of so my learning can be celebrated and added to my portfolio.

Acceptance Criteria

Students can tap a large, visible button labeled:

“I’m Proud of This!”

or equivalent child-friendly wording


Button available from:

Task pages

Quest pages

Showcase projects

Bounty completions


Trigger sends facilitator notification including:

Student name

Related quest/project

Timestamp




---

User Story 6.2 — Quick Celebration & Documentation Flow

As a facilitator, I want a fast and relationship-centered documentation process so I can celebrate student growth without disrupting momentum.

Acceptance Criteria

Notification opens a quick mobile-friendly workflow allowing facilitator to:

Take/upload photo

Record short observation

Add voice-to-text notes

Tag related learning domains

Save directly to portfolio


Entire process designed to take under 1–2 minutes.


---

User Story 6.3 — AI-Suggested Reflection Prompts for Facilitators

As a facilitator, I want simple curiosity-driven prompts so I can quickly gather meaningful context about student learning.

Acceptance Criteria

When documenting student work, platform provides optional prompts such as:

“What are you most proud of?”

“What was tricky about this?”

“What did you learn while making this?”

“What would you try next time?”

“Did anyone help you with this?”

“What part took the longest?”

“Why did you choose this project?”

“How did you solve problems while working?”


Prompts should:

Be age-appropriate

Encourage reflection without feeling evaluative

Support authentic conversation

As a facilitator, I want AI to help organize and contextualize student reflections so portfolio documentation remains meaningful but lightweight.

Acceptance Criteria

AI analyzes:

Facilitator observations

AI suggests:

Portfolio domain alignment

Creativity

Initiative

User Story 6.5 — Preserve Student Momentum During Documentation

As a facilitator, I want documentation to feel celebratory and conversational rather than evaluative so students stay excited and engaged in their learning.

Acceptance Criteria

System design prioritizes:

Quick interactions

Minimal required fields

Mobile-first documentation

Natural conversation flow

Celebration over assessment


Documentation process should:

Avoid requiring formal “pass offs”

Avoid interrupting active engagement

Encourage students to continue working after sharing accomplishments
